<td><p><strong>Note d’application</strong></p></td>
<td class="en">
<p>This class comprises physical objects (printed books, scores, CDs, DVDs, CD-ROMS, etc.) that were produced by (P186i) an industrial process involving a given instance of F3 Manifestation. As a result, all the instances of F5 Item associated with a given instance of F3 Manifestation are expected to carry the content defined in that instance of F3 Manifestation, although some or even all of them may happen to carry a content that significantly differs from it, due to either an accident in the course of industrial production, or subsequent physical modification or degradation.</p>
<p>An instance of F5 Item that consists of a physical object or set of objects with clear physical boundaries is also an instance of E22 Human-Made Object. An instance of F5 Item that is stored on a part of a larger physical support (such as an electronic file among others on a disc) can also be considered to be an instance of E25 Human-Made Feature.</p>
<p>The notion of F5 Item is only relevant with regard to the production process, from a bibliographic point of view. The physical units managed by cultural heritage institutions in their holdings are a distinct notion: a unit of holdings certainly can be equal to an instance of F5 Item, but it also can be either “bigger” than one (e.g., when two instances of F5 Item are bound together (in the case of printed books)), or “smaller” than one (e.g., for incomplete holdings, such as when only one CD from a two-CD set is held). From an operational point of view, cultural heritage institutions typically do <em>not</em> manage instances of F5 Item, but physical holdings units, instances of E19 Physical Object, although for libraries in most cases this is not significant because each item corresponds with a single unit. When this is not the case, the linkage between items and the units relevant for collection management can be recorded through the <em>P46 is composed of (forms part of)</em> property between instances of F5 Item and instances of E19 Physical Object. If needed, an instance of E19 Physical Object can be typed as a unit through the <em>P2 has type (is type of)</em> property.</p>
</td>
<td>
<p>Cette classe comprend les choses matérielles (p. ex. les livres imprimés, les partitions, les disques audio-numériques, les DVD, les cédéroms, etc.) produites (<code class="language-plaintext highlighter-rouge">P186i_est_produit_par</code>) grâce à des activités industrielles impliquant une instance de <code class="language-plaintext highlighter-rouge">F3_Manifestation</code>. Ainsi, il est attendu des instances de <code class="language-plaintext highlighter-rouge">F5_Item</code> associées à une instance de <code class="language-plaintext highlighter-rouge">F3_Manifestation</code> qu’elles soient porteuses du contenu de cette manifestation. Il est cependant possible que certaines, voire que toutes les instances de <code class="language-plaintext highlighter-rouge">F5_Item</code> concernées, soient porteuses de contenus significativement différents, que ce soit en raison d’imprévus survenus dans le cadre de l’activité industrielle ou en raison de modifications ou de dégradations matérielles subséquentes à celle-ci. </p>
<p>Une instance de <code class="language-plaintext highlighter-rouge">F5_Item</code> consistant en a) une chose matérielle ou b) un ensemble de celles-ci ayant des délimitations matérielles claires est aussi une instance de <code class="language-plaintext highlighter-rouge">E22_Objet_élaboré_par_l’humain</code>. Une instance de <code class="language-plaintext highlighter-rouge">F5_Item</code> contenue dans un élément d’un support matériel plus large (comme un fichier électronique parmi d’autres sur un disque) peut aussi être considérée comme une instance de <code class="language-plaintext highlighter-rouge">E25_Caractéristique_élaborée_par_l’humain</code>. </p>
<p>D’un point de vue bibliographique, la notion de <code class="language-plaintext highlighter-rouge">F5_Item</code> n’est pertinente qu’au regard du processus de production. Les éléments matériels relevant des collections d’institutions culturelles et patrimoniales diffèrent entièrement : un objet de collection peut équivaloir à une instance de <code class="language-plaintext highlighter-rouge">F5_Item</code>, mais il peut aussi le « comprendre » (p. ex. lorsque deux instances de <code class="language-plaintext highlighter-rouge">F5_Item</code> sont reliées, comme c’est le cas de livres imprimés) ou le « dépasser » (p. ex. dans le cas d’éléments incomplets de la collection, comme lorsqu’un seul disque audio-numérique d’un ensemble de deux se trouve dans une collection). D’une perspective opérationnelle, les institutions culturelles et patrimoniales n’ont habituellement pas dans leurs collections des instances de <code class="language-plaintext highlighter-rouge">F5_Item</code>, mais plutôt des objets de collection matériels, instances de <code class="language-plaintext highlighter-rouge">E19_Objet_matériel</code> ; pour les bibliothèques, dans la plupart des cas, ceci n’est pas un enjeu déterminant, car chaque item correspond à un élément unique. Lorsque ce n’est pas le cas, la relation entre les items et les éléments pertinents à la gestion de la collection peut être enregistrée grâce à la propriété <code class="language-plaintext highlighter-rouge">P46_est_composé_de (fait_partie_de)</code> entre des instances de <code class="language-plaintext highlighter-rouge">F5_Item</code> et de <code class="language-plaintext highlighter-rouge">E19_Objet_matériel</code>. Lorsque c’est nécessaire, une instance de <code class="language-plaintext highlighter-rouge">E19_Objet_matériel</code> peut aussi être catégorisée en tant qu’élément grâce à la propriété <code class="language-plaintext highlighter-rouge">P2_a_pour_type (est_le_type_de)</code>. </p>
</td>
